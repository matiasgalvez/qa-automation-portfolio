# AI-Assisted QA and LLM Testing

This section demonstrates my experience using AI tools as part of a modern QA workflow and testing AI-powered chat experiences in web applications.

My AI-related QA experience is divided into two areas:

1. AI-assisted QA engineering using tools such as Claude Code, Codex and ChatGPT.
2. LLM/chatbot testing for virtual assistants and conversational web applications.

## AI-Assisted QA Engineering

I use AI agents to support QA engineering tasks such as test generation, code review, automation maintenance, debugging, documentation and framework improvements.

AI tools help accelerate the work, but I keep ownership of the QA strategy, test design, assertions, coverage decisions and final validation.

## Human QA Review

AI-generated output is not accepted blindly.

I review:

- Test relevance.
- Assertion quality.
- Code maintainability.
- Selector reliability.
- Edge case coverage.
- Security or data exposure risks.
- False positives and false negatives.
- Alignment with acceptance criteria.

## LLM Chatbot Testing

I also understand how to test LLM-powered chat experiences and virtual assistants inside web applications.

This type of testing is different from traditional deterministic testing because the same input may not always return the exact same response.

Instead of only checking exact text, the QA strategy should validate:

- Intent understanding.
- Response relevance.
- Context retention.
- Safety behavior.
- Hallucination risk.
- Escalation paths.
- Error handling.
- Tone and brand alignment.
- Compliance with product rules.
- Conversation flow quality.
